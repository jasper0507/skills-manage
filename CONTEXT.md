# Skills Manage

本机 agent skills 的统一管理：盘点、分类与选用；跨 CLI 可达为次要能力。

**v1 范围**：只做 **分类工作台**（扫描本机 Skill + 桌面式 Web 归类 + 用户级中央索引 + **图标级**回收站）。**不做 Skill 本体隔离 / 真删 / 30 天删包**（安全与半残风险未收口前不进 v1）。**挑选器 / 入口 Skill / fzf / 调用串** 为后续模块，可与工作台同属一个产品，但不在 v1。

**原型结论（2026-07-14）**：`prototypes/workbench-desktop/` 抛弃型原型已基本符合产品意图；下列语言与规则以该验证为准，实现时应遵守，而非重新发明交互。

## Language

### 核心对象

**Skill**:
符合 agentskills 约定的本机目录包，以 `SKILL.md` 为入口；指机器上已存在的技能，不指市场可下载条目。
_Avoid_: 插件（plugin）、命令（command）、规则（rule）、MCP server

**清单（Inventory）**:
对本机已发现 Skill 的统一视图；不负责从远端市场拉取。
_Avoid_: 市场、注册表、目录站

**占位（Placeholder）**:
工作台上代表某个 Skill 的 **快捷方式图标**；指向 Skill 身份（realpath），不是文件副本。用户拖的是占位。
_Avoid_: 复制 Skill 目录、把路径字符串当唯一 UI

**Skill 身份**:
以规范化 **realpath** 唯一标识；多软链合并为同一 Skill。
_Avoid_: 仅目录名、仅 frontmatter name、内容哈希、UUID（v1）

**成员关系（Membership）**:
某枚占位是否属于某一 **普通盒** 或 **组合盒隔间**；**一枚占位至多一处**。盒内「有哪些图标」以成员关系为唯一真相。同一 Skill 要出现在多个盒中 → **复制**出多枚占位（多身份入口），不是一枚占位挂多个容器。
_Avoid_: 一枚占位多处挂载、成员关系与「在盒位置」各记一份互相打架

**占位安置（Placement）**:
占位不在盒内时的落点：要么在 **桌面图标网格**（有行列），要么在 **回收站站内**（图标级进站）。在盒内时，安置由 **成员关系** 表达，不再平行维护一套「在某盒」的第二真相。工作台视图可把成员关系呈现为「在盒中」，供人阅读。
_Avoid_: 盒内再存一份可与成员关系冲突的位置记录、用位置字段反推盒内名单

### 桌面隐喻（已验证）

**桌面（Desktop）**:
分类工作台主表面，对齐 **Windows 桌面 + Coodesker（酷呆桌面）盒子**：
- Skill 占位 = 可拖放的 **快捷方式图标**（受 **图标网格** 约束）
- 盒子 = 可展示内容的 **大型文件夹 / 整理盒**
- 回收站 = **系统快捷方式图标**（可拖、可入盒；不可复制/剪切/删除该图标本身）；语义见「图标级回收站」

_Avoid_: 纯表单后台、无桌面隐喻的标签管理器

**默认桌面布局**:
- **回收站** 默认在图标网格 **第 1 行第 1 列**
- **待整理 Skill**（未入盒占位）在默认 **一屏视口网格**（约 12 列 × 8 行）内 **行优先** 填空：先向右再换行，不从第一列无限向下堆叠
- 用户之后可拖走；重扫新增 Skill 填 **空位**，不重排已有坐标；视口满后才向下溢出

_Avoid_: 回收站默认扔在角落无约定、新 skill 打乱用户已摆布局、初始一列纵向长龙超出一屏

**图标网格**:
图标落点吸附到固定格子；**正常不允许两枚 Skill 图标占同一格**。
_Avoid_: 任意像素重叠堆叠

**桌面碰撞规则**:
1. **图标 → 图标**（同一网格格）：自动创建 **普通盒子**，默认名「普通盒子」+ 序号（可双击改名），相关占位收入该盒。  
2. **图标 → 盒子**（拖动相交）：占位加入该盒 **当前标签/隔间**。  
3. **盒子 → 桌面**：盒子矩形 **不得压住** 桌面 Skill 图标（避让/拒绝压住）。  
4. **图标 → 回收站**：走 **图标级** 进站（见回收站）；**不得**触发 Skill 本体删除。  

_Avoid_: 静默重叠、盒子压住图标无反馈

### 盒子与标签

**普通盒子（Simple box）**:
恰好 **一个标签**（兼显示名）；单一容器。可拖动位置；无显式「删除」按钮，**右键 → 删除盒子**。
_Avoid_: 普通盒多标签

**组合盒子（Composite box）**:
**盒标题**（主题，如「Go 开发」）+ **多个隔间标签**；每隔间独立容器。**悬停标签**切换当前隔间。
_Avoid_: 无标题仅靠标签拼接、多标签共用一个未分区列表

**组盒规则**:
- 普通 → 普通：合成组合盒（双方标签成两隔间；默认标题来自目标盒）  
- 普通 → 组合：追加隔间  
- 隔间拖出：变为普通盒  
- 组合盒 **只剩一个隔间**：立即 **退化为普通盒**  
- 禁止组合 → 组合整盒合并  

_Avoid_: 单隔间仍保持组合形态、拖出丢占位

**隔间（Compartment）**:
组合盒内某一标签下的占位集合；投放目标是 **当前可见隔间**。
_Avoid_: 与整盒混成一个列表

**标签（Tag）** / **盒标题（Box title）**:
标签：普通盒一个；组合盒每个隔间一个。盒标题：仅组合盒主题名，≠ 隔间标签。  
**双击**普通盒标签名 / 组合盒隔间标签 / 组合盒标题 → 重命名。  
自动组盒默认名「普通盒子N」，等待用户改名。
_Avoid_: 只能删建改名、用隔间标签顶替主题标题

**盒内展示**:
盒内必须以 **图标**（可辅以列表信息）显式展示已收纳 Skill，用户能看到刚拖入了什么。
_Avoid_: 拖入后盒内空白

**删盒**:
盒内全部占位 **回桌面网格空位**（含曾在盒内的回收站图标则回桌面）。不删 Skill 本体；占位不进回收站。
_Avoid_: 删盒导致占位消失或触发 Skill 本体删除

### 剪贴板与多选

**复制 / 剪切 / 粘贴**:
Skill **右键**：复制、剪切、删除（**图标级进站**，规则同回收站）、**多选**（进入时 **默认已选当前**；可「取消多选」）。  
桌面空白 / 盒内空白 **右键**：粘贴；桌面空白另可 **创建新盒子 → 普通 / 组合**。  
无常驻复制按钮。剪切/粘贴与复制只动占位，不碰磁盘包。复制用于 **同一 Skill 多处归类**（多枚占位、同一身份）。
_Avoid_: 工具栏多选/复制为主入口、把「删除」理解成卸载 Skill

**多选（Multi-select）**:
右键开启；批量拖入盒、批量搬移、或批量 **图标级** 进站。进站 **按身份** 约束：若某身份在本批全部进站后活占位会归零，则 **该身份本批跳过**（其它身份仍可进站）；禁止静默删光某 Skill 的工作台入口。
_Avoid_: 仅能单枚操作、一批因一个身份失败而全部回滚、批量进站后某身份零入口

### 图标级回收站（v1）

**回收站（Recycle bin）**:
- 桌面系统图标；默认 **(行1, 列1)**  
- **可拖动**到桌面网格空位、**可入盒**；**不可**复制/剪切/删除该图标本身  
- 站内是 **已移除的占位（快捷方式）**，不是隔离中的 Skill 目录  
- **还原**：占位回到桌面网格空位  
- **清空**：仅丢弃站内占位记录；**不** rename、**不** `rm` Skill 包；成品无工具栏清空，入口为右键回收站  
- 拖入 / 删除前可确认（文案须表明只影响工作台图标）

**进站规则（图标级）**:
- 去掉的是 **占位**，磁盘 Skill 包始终保留在扫描根内、保持可被 agent 加载  
- 若某身份在「桌面 + 盒内」的 **活占位**（非站内）去掉后会变成 0 枚 → **拒绝进站**（至少保留一枚工作台入口）  
- 多占位时：可进站多余副本，便于清理复制残留；多盒归类靠复制实现  

**本体删除 / 隔离 / 真删**:
**非 v1。** 不做同文件系统隔离树、不做 30 天删包、不暴露 purge 包 API。后续若做，另开设计并见 `docs/research/safe-recycle-bin-deletion.md`。
_Avoid_: 把清空回收站做成删本机 skill、最后一枚占位进站后磁盘包消失、仅 index 标记而宣称「已删除 skill」、v1 半套 quarantine

### 扫描与索引

**扫描根（Scan root）**:
默认用户级与当前仓库常见 skills 路径（如 `~/.agents/skills`、`~/.claude/skills`、`~/.codex/skills`、`~/.grok/skills` 及项目内对应目录）；可配置；bundled/system 默认忽略。
_Avoid_: 只扫单一路径且不可配置

**重扫增量**:
不重排已有桌面/盒子布局；新 Skill 出现在网格 **空位**。
_Avoid_: 每次全量重排

**中央索引（Index）**:
用户级 **单文件 JSON**（如 `~/.config/skills-manage/index.json`）：Skill 身份与名称缓存、盒子/隔间/标题、**成员关系**（盒/隔间内的占位 id 列表）、桌面网格坐标、回收站系统图标位置、站内占位标记。不写 Skill 包内。原子写（temp→rename）。本机 skill 数量有限，v1 不上 SQLite。v1 **不**持久化本体隔离/quarantine 删包元数据；读到旧版隔离字段可忽略并剥离。盒内成员以成员关系为准，不为盒内占位再存一套可冲突的「在盒」位置。
_Avoid_: sidecar 改 SKILL.md、v1 默认 SQLite、成员关系与盒内位置双源真相

### 产品与实现

**分类工作台（Taxonomy workbench）**:
v1 交付物：一条 CLI 起本机 HTTP 并打开浏览器；进程随命令结束（非常驻 daemon）。
_Avoid_: v1 常驻后台服务

**工具（Product）**:
一个聚合产品：v1 = 工作台；后续并入挑选器模块，不另起无关产品。
_Avoid_: 两个互不相干的独立产品

**挑选器 / 入口 Skill / 调用串**:
规划中，**非 v1**（会话内选用、同终端 fzf 等意向保留在研究笔记与早期讨论中，实现前可再开一轮细化）。
_Avoid_: 当作 v1 必交付

**跨 CLI 同步**:
附加能力，非 v1 主路径（可用软链等过渡）。
_Avoid_: v1 主做安装市场

**实现栈（v1）**:
**Go** 单二进制：CLI、扫描、索引、图标级回收站、内嵌静态前端。运行时不依赖 Node/Python。v1 **不含** Skill 包隔离/真删子系统的产品路径。
_Avoid_: 必须本机装 Node 才能跑、v1 Electron

**前端**:
交互以已验证的桌面原型为准；实现阶段再定 embed 用的 UI 技术（React/Svelte/等），**不**为框架阻塞领域。成品 **无「重置」测试按钮**。
_Avoid_: 无视已验证交互另起炉灶、面向用户的一键清空布局
