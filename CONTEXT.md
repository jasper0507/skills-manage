# Skills Manage

本机 agent skills 的统一管理：盘点、分类与选用；跨 CLI 可达为次要能力。

**v1 范围**：只做 **分类工作台**（扫描本机 Skill + 桌面式 Web 归类 + 用户级中央索引 + 回收站真删生命周期）。**挑选器 / 入口 Skill / fzf / 调用串** 为后续模块，可与工作台同属一个产品，但不在 v1。

**原型结论（2026-07-14）**：`prototypes/workbench-desktop/` 抛弃型原型已基本符合产品意图；下列语言与规则以该验证为准，实现时应遵守，而非重新发明交互。

## Language

### 核心对象

**Skill**:
符合 agentskills 约定的本机目录包，以 `SKILL.md` 为入口；指机器上已存在的技能，不指市场可下载条目。
_Avoid_: 插件（plugin）、命令（command）、规则（rule）、MCP server

**清单（Inventory）**:
对本机已发现 Skill 的统一视图；不负责从远端市场拉取。
_Avoid_: 市场、注册表、目录站

**占位（Placeholder）**:
工作台上代表某个 Skill 的 **快捷方式图标**；指向 Skill 身份（realpath），不是文件副本。用户拖的是占位。
_Avoid_: 复制 Skill 目录、把路径字符串当唯一 UI

**Skill 身份**:
以规范化 **realpath** 唯一标识；多软链合并为同一 Skill。
_Avoid_: 仅目录名、仅 frontmatter name、内容哈希、UUID（v1）

### 桌面隐喻（已验证）

**桌面（Desktop）**:
分类工作台主表面，对齐 **Windows 桌面 + Coodesker（酷呆桌面）盒子**：
- Skill 占位 = 可拖放的 **快捷方式图标**（受 **图标网格** 约束）
- 盒子 = 可展示内容的 **大型文件夹 / 整理盒**
- 回收站 = **系统快捷方式图标**（可拖、可入盒；不可复制/剪切/删除该图标本身）

_Avoid_: 纯表单后台、无桌面隐喻的标签管理器

**默认桌面布局**:
- **回收站** 默认在图标网格 **第 1 行第 1 列**
- **待整理 Skill**（未入盒占位）在默认 **一屏视口网格**（约 12 列 × 8 行）内 **行优先** 填空：先向右再换行，不从第一列无限向下堆叠
- 用户之后可拖走；重扫新增 Skill 填 **空位**，不重排已有坐标；视口满后才向下溢出

_Avoid_: 回收站默认扔在角落无约定、新 skill 打乱用户已摆布局、初始一列纵向长龙超出一屏

**图标网格**:
图标落点吸附到固定格子；**正常不允许两枚 Skill 图标占同一格**。
_Avoid_: 任意像素重叠堆叠

**桌面碰撞规则**:
1. **图标 → 图标**（同一网格格）：自动创建 **普通盒子**，默认名「普通盒子」+ 序号（可双击改名），相关占位收入该盒。  
2. **图标 → 盒子**（拖动相交）：占位加入该盒 **当前标签/隔间**。  
3. **盒子 → 桌面**：盒子矩形 **不得压住** 桌面 Skill 图标（避让/拒绝压住）。  
4. **图标 → 回收站**：走删除/回收流程（见回收站）。  

_Avoid_: 静默重叠、盒子压住图标无反馈

### 盒子与标签

**普通盒子（Simple box）**:
恰好 **一个标签**（兼显示名）；单一容器。可拖动位置；无显式「删除」按钮，**右键 → 删除盒子**。
_Avoid_: 普通盒多标签

**组合盒子（Composite box）**:
**盒标题**（主题，如「Go 开发」）+ **多个隔间标签**；每隔间独立容器。**悬停标签**切换当前隔间。
_Avoid_: 无标题仅靠标签拼接、多标签共用一个未分区列表

**组盒规则**:
- 普通 → 普通：合成组合盒（双方标签成两隔间；默认标题来自目标盒）  
- 普通 → 组合：追加隔间  
- 隔间拖出：变为普通盒  
- 组合盒 **只剩一个隔间**：立即 **退化为普通盒**  
- 禁止组合 → 组合整盒合并  

_Avoid_: 单隔间仍保持组合形态、拖出丢占位

**隔间（Compartment）**:
组合盒内某一标签下的占位集合；投放目标是 **当前可见隔间**。
_Avoid_: 与整盒混成一个列表

**标签（Tag）** / **盒标题（Box title）**:
标签：普通盒一个；组合盒每个隔间一个。盒标题：仅组合盒主题名，≠ 隔间标签。  
**双击**普通盒标签名 / 组合盒隔间标签 / 组合盒标题 → 重命名。  
自动组盒默认名「普通盒子N」，等待用户改名。
_Avoid_: 只能删建改名、用隔间标签顶替主题标题

**盒内展示**:
盒内必须以 **图标**（可辅以列表信息）显式展示已收纳 Skill，用户能看到刚拖入了什么。
_Avoid_: 拖入后盒内空白

**删盒**:
盒内全部占位 **回桌面网格空位**；若回收站图标在盒内，亦回桌面。不删 Skill 本体。
_Avoid_: 删盒导致占位消失

### 剪贴板与多选

**复制 / 剪切 / 粘贴**:
Skill **右键**：复制、剪切、删除、**多选**（进入时 **默认已选当前**；可「取消多选」）。  
桌面空白 / 盒内空白 **右键**：粘贴；桌面空白另可 **创建新盒子 → 普通 / 组合**。  
无常驻复制按钮。
_Avoid_: 工具栏多选/复制为主入口

**多选（Multi-select）**:
右键开启；批量拖入盒或回收站。
_Avoid_: 仅能单枚操作

### 回收站与删除

**回收站（Recycle bin）**:
- 桌面系统图标；默认 **(行1, 列1)**  
- **可拖动**到桌面网格空位、**可入盒**；**不可**复制/剪切/删除该图标  
- **清空**仅：右键回收站 → 清空当前回收站（成品无工具栏清空）  
- 拖入 Skill 占位须确认  

**占位删除 vs 本体删除**:
去掉一枚占位通常 **只删图标**。仅当该 Skill **全局只剩最后一枚占位** 再删除/进站时，才进入 **本体** 删除生命周期。
_Avoid_: 多占位时误删磁盘本体

**隔离（Quarantine / rename isolation）**:
本体进站后用 **同文件系统 rename** 离开 live 扫描名（每扫描根下 `.skills-manage-trash/<id>/`）。进站/还原为元数据操作，不整树拷贝内容。跨盘不作为默认。  
保留 **30 天** 可还原；到期或清空 → **真删** 隔离目录。  
详见 `docs/research/safe-recycle-bin-deletion.md`。
_Avoid_: 仅索引标记而 live 路径仍可被 agent 加载、默认跨盘 copy

**删除安全**:
确认并展示路径；只删扫描认定且 realpath 在 **扫描根内** 的 skill 包；拒绝 bundled/system/根外/危险软链；同一 realpath 单一生命周期；可审计；purge 状态机防半残。
_Avoid_: 任意路径 rm

### 扫描与索引

**扫描根（Scan root）**:
默认用户级与当前仓库常见 skills 路径（如 `~/.agents/skills`、`~/.claude/skills`、`~/.codex/skills`、`~/.grok/skills` 及项目内对应目录）；可配置；bundled/system 默认忽略。
_Avoid_: 只扫单一路径且不可配置

**重扫增量**:
不重排已有桌面/盒子布局；新 Skill 出现在网格 **空位**。
_Avoid_: 每次全量重排

**中央索引（Index）**:
用户级 **单文件 JSON**（如 `~/.config/skills-manage/index.json`）：身份、盒子/隔间/标题、占位引用、网格坐标、回收站与隔离元数据。不写 Skill 包内。原子写（temp→rename）。本机 skill 数量有限，v1 不上 SQLite。
_Avoid_: sidecar 改 SKILL.md、v1 默认 SQLite

### 产品与实现

**分类工作台（Taxonomy workbench）**:
v1 交付物：一条 CLI 起本机 HTTP 并打开浏览器；进程随命令结束（非常驻 daemon）。
_Avoid_: v1 常驻后台服务

**工具（Product）**:
一个聚合产品：v1 = 工作台；后续并入挑选器模块，不另起无关产品。
_Avoid_: 两个互不相干的独立产品

**挑选器 / 入口 Skill / 调用串**:
规划中，**非 v1**（会话内选用、同终端 fzf 等意向保留在研究笔记与早期讨论中，实现前可再开一轮细化）。
_Avoid_: 当作 v1 必交付

**跨 CLI 同步**:
附加能力，非 v1 主路径（可用软链等过渡）。
_Avoid_: v1 主做安装市场

**实现栈（v1）**:
**Go** 单二进制：CLI、扫描、索引、隔离/真删、内嵌静态前端。运行时不依赖 Node/Python。
_Avoid_: 必须本机装 Node 才能跑、v1 Electron

**前端**:
交互以已验证的桌面原型为准；实现阶段再定 embed 用的 UI 技术（React/Svelte/等），**不**为框架阻塞领域。成品 **无「重置」测试按钮**。
_Avoid_: 无视已验证交互另起炉灶、面向用户的一键清空布局
