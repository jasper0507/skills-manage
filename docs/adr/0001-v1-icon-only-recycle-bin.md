# v1 回收站仅为图标级软回收（禁止 Skill 本体删除）

分类工作台需要桌面式回收站隐喻，以便用户清理「复制出来的多余占位」，并支持同一 Skill 多盒归类。完整「本体隔离 + 30 天真删」路径（scan-root 下 rename、`RemoveAll`、崩溃半残）在 v1 安全边界未收口，**不进入 v1**。

**决定：** v1 采用 **图标级软回收（R2）**：

- 桌面保留回收站系统图标；占位可进站、可还原到桌面空位；清空回收站 **只丢弃占位记录**。
- **禁止**某 Skill 身份的最后一枚活占位进站（工作台对该 Skill 至少保留一枚入口）。
- **禁止**任何路径对 Skill 包做隔离 rename 或真删；不提供本体删除生命周期。

调研 `docs/research/safe-recycle-bin-deletion.md` 留给后续「真删模块」；现行语言见 `CONTEXT.md`。
